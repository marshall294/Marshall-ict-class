Year 3 Computing — Week 4

Topic: Computing Systems and Networks – Connecting Computers

Teacher: Marshall Jacob | Duration: 45 minutes

Learning Objective

To explain how a computer network can be used to share information.

Success Criteria

  • ✔️ I can discuss why we need a network switch.
  • ✔️ I can explain how messages are passed through multiple connections.
  • ✔️ I can recognise different connections (wired, wireless, cables).

Teacher Notes — Lesson Flow

  1. Starter (5 mins): Show pictures of two computers and ask: "How can they share a message?" Brainstorm answers (cable, Wi-Fi, Bluetooth).
  2. Main Activity (20 mins): Explain what a switch does: it helps direct messages to the right computer. Use a simple diagram (board or screen) or a live role play with pupils as "computers".
  3. Practical Activity (12 mins): Pupils form small groups with cards/strings to make a mini network. Send a paper message from one computer to another via the switch (a pupil acting as the switch decides where to pass it).
  4. Plenary (8 mins): Recap the path a message takes and ask volunteers to explain why switches help networks work well.

Key Concepts

  • Network: Two or more computers connected so they can share information.
  • Switch: A device that connects many computers and sends messages to the correct place.
  • Connection types: Wired (Ethernet cable), wireless (Wi-Fi), and Bluetooth.
  • Message passing: Messages can travel through several devices before reaching their destination.

Examples & Simple Diagrams

Example 1: Computer A sends a document to Computer B → message goes A → switch → B.
Example 2: If B is busy, switch sends the message to C instead (switch decides the best path).
Example 3: Wireless example — a tablet uses Wi-Fi to ask for a picture on the shared printer.

Class Activity — Role Play Network

Materials: paper "messages", name cards (Computer A, B, C), string to show connections.

  1. Give each group 3–5 pupils a role (computers, switch, router).
  2. Pupils pass a paper message through the network following the connections.
  3. The pupil acting as the switch directs messages to the correct computer.

Learning goal: see how messages travel and how switches help choose where a message goes.

Assessment — Quick Check

Q1: What does a network switch do?

Q2: Which is a wired connection?

Q3: How might a message travel from A to D?

Q4: Which is an example of a wireless connection?

Q5: Why do networks help schools?

Q6: True or False: A message always goes in a straight line — it may pass through many devices.

Q7: Name one connection you recognise (pupil answer).
Use this as a short oral check in class — listen for answers like "Wi-Fi", "Ethernet", "Bluetooth".

Extension / Home Learning

  • Draw your own network with at least three devices and label the switch and connections.
  • Write one sentence explaining why switches are useful.
