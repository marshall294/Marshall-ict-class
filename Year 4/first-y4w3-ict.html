Learning Objective

To outline how websites can be shared via the World Wide Web (WWW) and understand how we access and view information online.

Success Criteria

  • ✔️ I can describe how to access websites on the WWW.
  • ✔️ I can describe where websites are stored when uploaded to the WWW.
  • ✔️ I can explain the types of media that can be shared on the WWW.

Key Notes and Definitions

  • World Wide Web (WWW): A collection of web pages that you can access using the internet.
  • Website: A group of connected web pages created by people or organizations.
  • Web Server: A computer that stores websites and sends them to your device when requested.
  • Web Browser: A program (like Chrome, Safari, or Edge) used to view websites.
  • URL (Uniform Resource Locator): The unique address of a website, e.g., https://www.bbc.co.uk.
  • Hyperlink: A piece of text or image that you can click to open another webpage.
  • Media: The different forms of content shared online — text, images, videos, sounds, and animations.

How the World Wide Web Works

When you type a website address (URL) into your browser, your computer sends a request through the internet to a web server. The server then sends back the website’s files, such as text, pictures, or videos, which your browser displays on your screen.

Example: When you type www.nationalgeographic.com, the request travels to the web server storing National Geographic’s website, which sends back all the content for you to see.

Where Are Websites Stored?

Websites are stored on web servers. These are special computers that stay online 24/7, allowing anyone in the world to access the sites at any time. Big companies may have their own servers, while smaller websites use hosting services that rent out space on shared servers.

What Can Be Shared on the WWW?

  • Text: Articles, blogs, and stories.
  • Images: Photos and illustrations.
  • Videos: Educational clips, movies, and animations.
  • Sound: Music, podcasts, and sound effects.
  • Interactive media: Games and quizzes that respond to user input.

Examples and Activities
